Relatório Técnico: Introdução aos Sistemas Operacionais
1. Objetivo Este relatório tem como objetivo apresentar uma síntese detalhada da Unidade 1 do livro "Sistemas Operacionais", abordando a definição, o histórico, as arquiteturas e as características dos principais sistemas operacionais do mercado, como Windows e Linux. Busca-se compreender como esse software atua como o elo essencial entre o hardware, o software e o usuário.
2. Introdução Um sistema operacional (SO) é definido como um conjunto de programas cuja função é gerenciar os recursos de hardware (como processador, memória e periféricos) e oferecer uma interface para que o usuário interaja com a máquina de forma eficiente. Segundo os conceitos fundamentais, o SO atua em duas frentes principais:
* Gerenciador de recursos: Controla quem usa qual recurso e resolve conflitos entre programas, garantindo que o hardware seja utilizado de forma compartilhada no tempo e no espaço.
* Máquina Estendida (Abstração): Esconde a complexidade técnica dos componentes físicos do computador, apresentando uma interface gerenciável e amigável para o programador e o usuário.
3. Métodos (Desenvolvimento e Evolução)
3.1 Histórico e Gerações A evolução dos sistemas operacionais é categorizada em quatro gerações principais, acompanhando o avanço do hardware:
* 1ª Geração (1945-1955): Máquinas a válvulas, sem sistemas operacionais; a programação era feita manualmente em painéis.
* 2ª Geração (1955-1965): Introdução dos transistores e sistemas em lote (batch), onde os trabalhos eram processados em sequência via fitas magnéticas.
* 3ª Geração (1965-1980): Surgimento dos circuitos integrados e da multiprogramação, permitindo que vários programas compartilhassem a memória.
* 4ª Geração (1980-Hoje): Era dos computadores pessoais com interfaces gráficas, redes de computadores e sistemas distribuídos.
3.2 Estrutura e o Núcleo (Kernel) O Kernel é o componente central que executa as funções vitais, como o gerenciamento de processos, memória e arquivos. Para garantir a segurança, o sistema opera em dois modos:
* Modo Usuário: Onde rodam as aplicações comuns, com privilégios restritos.
* Modo Kernel (ou Núcleo): Onde o SO tem acesso total ao hardware. A comunicação entre esses modos ocorre através de chamadas de sistema (system calls).
3.3 Arquiteturas e Classificação As arquiteturas variam conforme o projeto:
* Monolítica: Todos os componentes estão integrados em um único arquivo executável (ex: MS-DOS).
* Camadas: Organizado em níveis hierárquicos, onde cada camada oferece funções para a superior.
* Máquinas Virtuais: Criam réplicas do hardware real para rodar múltiplos sistemas independentes no mesmo equipamento.
* Sistemas Multiprogramáveis: Classificam-se em sistemas de tempo compartilhado (timesharing), onde a CPU é dividida em fatias de tempo (time-slice), e sistemas de tempo real, onde prazos de resposta são rígidos.
4. Características do Mercado: Linux vs. Windows O relatório destaca as principais diferenças entre as plataformas dominantes:
* Linux: Baseado no Unix, é um sistema de código aberto licenciado pela GPL, o que permite que qualquer usuário acesse ou modifique seu código. É conhecido pela flexibilidade e estabilidade, possuindo diversas distribuições (Debian, Ubuntu, Mint).
* Windows: Sistema proprietário da Microsoft. Ao contrário do Linux, seu código é restrito e o uso depende da aquisição de licenças pagas. Evoluiu de uma interface sobre o MS-DOS para uma arquitetura robusta baseada em micronúcleo (Windows NT).
5. Conclusão Conclui-se que o sistema operacional é o software mais importante de um computador, sendo indispensável para o aproveitamento do hardware moderno. A escolha entre um sistema como o Linux ou Windows deve considerar fatores como custo (licenciamento gratuito vs. pago), suporte (comunitário vs. corporativo) e a necessidade de flexibilidade no acesso ao código-fonte.
